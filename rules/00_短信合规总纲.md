# 金融行业短信模板合规审核规范（总纲）

> 本目录收录了各类金融行业短信模板的合规审核标准，适用于信贷、理财、支付、互联网金融等各类金融业务场景。

---

## 文档目录

| 编号 | 文档名称 | 适用范围 | 审核优先级 |
|------|---------|---------|-----------|
| 00 | 短信合规总纲（本文档） | 通用规则，所有短信类型 | 基础 |
| 01 | 验证码短信合规规范 | 验证码、动态口令、登录凭证类 | 高 |
| 02 | 营销短信合规规范 | 产品推广、活动宣传、优惠券发放等 | 高 |
| 03 | 催收短信合规规范 | 逾期催缴、欠款提醒、法律告知等 | **最高** |
| 04 | 权益通知短信合规规范 | 账户变动、交易通知、服务变更等 | 中 |

## 一、通用禁止条款（所有短信类型适用）

以下条款为**一票否决项**，任何类型的短信模板一旦违反，直接驳回：

### 🔴 绝对禁止

1. **伪造发送主体** — 不得冒充银行、监管机构、政府部门发送短信
2. **伪造法律文书** — 不得以法院、公安局、检察院等名义发送短信
3. **泄露用户隐私** — 不得在短信中明文展示用户身份证号、完整银行卡号等敏感信息
4. **嵌入钓鱼链接** — 不得嵌入未经安全审核的第三方链接
5. **保证收益承诺** — 任何情况下不得承诺"保证收益"、"稳赚不赔"
6. **恐吓威胁语言** — 不得威胁用户人身安全或公开用户隐私
7. **虚假内容** — 不得伪造交易记录、余额信息、活动内容

## 二、审核流程

```
提交申请 → 分类识别 → 按类审核 → 合规评级 → 发布/驳回
    │           │            │           │
    │           │            │           └── 🟢通过 / 🟡整改 / 🔴驳回
    │           │            │
    │           │            └── 对照对应类型规范逐项检查
    │           │
    │           └── 识别短信类型，对应到对应规范文档
    │
    └── 通过合规审核系统提交，附上模板内容及使用场景说明
```

### 审核时效

| 短信类型 | 标准审核时长 | 加急条件 |
|---------|------------|---------|
| 验证码短信 | 1个工作日 | — |
| 权益通知短信 | 1个工作日 | — |
| 营销短信 | 3个工作日 | 重大活动可申请加急至1日 |
| 催收短信 | **2个工作日** | 逾期率预警时可加急 |

## 三、合规评级说明

| 等级 | 标识 | 含义 | 处理方式 |
|------|------|------|---------|
| 合规 | 🟢 | 完全符合所有规范要求 | 批准发布 |
| 整改 | 🟡 | 部分条款不符合，需修改后复审 | 退回并标注问题 |
| 违规 | 🔴 | 存在严重违规内容或一票否决项 | 立即驳回，启动调查 |

---

## 四、链接报备统一要求

所有短信中的外部链接（含短链接）均须提前报备：

```
报备信息：
├── 链接URL
├── 链接类型（短链/直链）
├── 落地页标题
├── 落地页核心内容摘要（100字以内）
├── 跳转是否需要登录
├── 安全扫描结果
└── 审核状态（待审/已通过/已驳回）
```

**注**：短链服务商须为公司白名单内的合规服务商（详见IT部门发布的短链白名单）

## 五、短信分类判断指引

当一条短信涉及多个类型时，按以下优先级判定类型：

```
催收类 > 验证码类 > 营销类 > 权益通知类
```

**示例**：
- "您的验证码是123456，用于营销活动注册" → 归类为**验证码短信**
- "您有一笔待还欠款，现在还款可享折扣" → 归类为**催收短信**
- "您的积分可以兑换优惠券了" → 归类为**营销短信**

## 六、敏感时期规定

以下时期，所有短信（除验证码、安全通知外）发送须额外审批：

| 时期 | 要求 | 说明 |
|------|------|------|
| 3·15消费者权益日前后各一周 | 营销类短信暂停 | 避免曝光风险 |
| 两会期间 | 催收类短信降频 | 维护社会稳定 |
| 重大金融风险事件期间 | 全部暂停 | 响应监管要求 |
| 法定节假日前1天 | 营销类短信限量 | 避免投诉高峰 |

## 七、违规处理机制

| 违规级别 | 处理措施 |
|---------|---------|
| 单次轻微违规 | 通报批评，模板下架整改 |
| 多次违规或中等违规 | 暂停该业务线短信通道，整改后复审 |
| 严重违规（涉及用户资金损失） | 立即停止通道，赔偿用户损失，追究法律责任 |
| 涉及监管红线 | 同步报告监管机构，配合调查 |

## 八、模板管理规范

### 8.1 模板库建设

- 所有已审核通过的短信模板须录入模板库统一管理
- 模板库须包含：模板编号、版本号、适用业务线、审核状态、有效期
- 模板更新须重新提交审核

### 8.2 变量字段管理

模板中的变量字段须在文档中明确标注：

| 变量 | 格式要求 | 示例 |
|------|---------|------|
| `{name}` | 用户真实姓名 | 张三 |
| `{amount}` | 金额，保留2位小数 | 1,234.56 |
| `{date}` | 日期格式 YYYY-MM-DD | 2026-03-22 |
| `{phone}` | 脱敏手机号 | 138****5678 |
