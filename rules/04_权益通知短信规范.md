# 权益通知短信合规规范

> 本文档适用于所有以权益通知、账户变动、交易提醒、服务通知等为目的的短信审核。不包括：营销推广、催收催缴、验证码等类别。

> **典型场景包括**：账户登录通知、交易成功/失败通知、套餐变更通知、费率调整通知、积分变动通知、会员等级变更通知、到期续费提醒（不含营销）等。

---

## 一、基本原则

1. **准确性** — 通知内容必须与实际业务数据完全一致
2. **及时性** — 通知类短信须在触发事件后及时发送
3. **完整性** — 关键信息不得遗漏，不得使用"详见"省略重要内容
4. **纯中性** — 权益通知短信不得含有任何营销意图或诱导性内容

---

## 二、必含信息要素

| 要素 | 说明 | 备注 |
|------|------|------|
| 发送方名称 | 须使用实名，不得使用昵称或代号 | 如【XX金融】 |
| 通知类型 | 明确告知本次通知的业务类型 | 如"交易通知"、"登录提醒" |
| 核心信息 | 时间、金额、状态等业务核心数据 | 须准确无误 |
| 操作入口 | 告知用户如何进一步查询或操作 | 如客服电话、APP入口 |
| 退订说明 | 告知如何取消此类通知 | 须提供退订选项 |

---

## 三、链接规定（重点）

### 3.1 链接报备要求

权益通知类短信中的所有链接（无论长短）**必须**提前报备：

```
报备字段：
- 链接URL
- 链接标题（如"查看详情"）
- 链接指向页面截图/内容摘要
- 是否需要用户登录
- 审核状态
```

### 3.2 允许嵌入链接的场景

- 用户主动查询类通知（如"点击查看完整账单"）
- 安全验证类链接（如邮箱/手机解绑链接）
- 服务续期类通知（如"点击续费以保持服务"）

### 3.3 禁止嵌入链接的场景

- 营销推广类内容（应归入营销短信审核）
- 第三方支付/转账引导（存在钓鱼风险）
- 收集用户敏感信息的表单链接

### 3.4 链接安全要求

- 链接须为公司自有域名或经安全认证的短链
- 链接须通过安全扫描，无钓鱼/木马风险
- 短链跳转后须与承诺页面一致，不得中途劫持

## 四、各场景合规示例

### 4.1 账户登录通知

```
【XX金融】您的账号于 2026-03-22 14:30 在新设备登录，设备：小米14，地点：上海。
如果不是您本人操作，请立即联系客服：400-XXX-XXXX 或在APP中冻结账户。
```

### 4.2 交易变动通知

```
【XX金融】您的账户于 03-22 15:00 完成提现，金额：¥5,000.00，
预计1-3个工作日到账，请注意查收。账户余额：¥12,345.67。
如有疑问请致电：400-XXX-XXXX
```

### 4.3 套餐/服务变更通知

```
【XX金融】您的会员等级已于03-01变更为"黄金会员"，有效期至2027-03-01。
当前享有：专属客服、优先处理、8折提现优惠。退订此类通知回复TD
```

### 4.4 费率调整通知

```
【XX金融】温馨提示：您的账户服务费将于04-01起调整为2元/月，
原费率1元/月。详情查看：https://xx.com/fees
如有异议请于03-31前联系客服：400-XXX-XXXX
```

## 五、禁止事项

| 禁止类型 | 违规示例 | 说明 |
|---------|---------|------|
| 虚假信息 | 捏造不存在的交易记录 | 严重违法 |
| 隐瞒关键条款 | "服务调整，详见协议"（实际收费提高） | 欺诈 |
| 诱导营销 | "限时开通VIP，仅需99元"（混入通知短信） | 违规营销 |
| 强制绑定 | "点击此链接必须绑定银行卡" | 强制搭售 |
| 泄露隐私 | "您的欠款XXX元已逾期"（非催收场景） | 信息泄露 |
| 伪装主体 | 冒充银行发送通知 | 伪造身份 |

## 六、退订管理

- 权益通知类短信属于用户已订阅的服务，**默认允许发送**
- 但用户退订后须立即停止发送（不得超过24小时）
- 部分高敏感通知（如登录异常、账户冻结）**不可退订**，须在短信中注明"此类通知不可关闭"
- 退订后再次发送相同类型通知须间隔至少30天

## 八、审核检查清单

- [ ] 发送主体名称是否真实准确
- [ ] 通知内容是否与实际业务数据一致
- [ ] 核心信息（金额、时间、状态）是否准确无误
- [ ] 链接是否已报备并通过安全审核
- [ ] 是否存在营销推广内容混入
- [ ] 是否存在隐瞒/省略关键信息
- [ ] 是否提供有效的客服联系方式
- [ ] 是否包含退订说明（或注明不可退订原因）
- [ ] 链接落地页是否与承诺内容一致

---

## 九、合规等级

| 等级 | 说明 | 处理方式 |
|------|------|---------|
| 🟢 合规 | 完全符合规范要求 | 通过 |
| 🟡 整改 | 信息不完整/链接未报备 | 退回修改后复审 |
| 🔴 违规 | 虚假信息/隐瞒关键条款/冒充主体 | 立即驳回并启动调查 |

---

## 十、模板库管理

建议建立标准化权益通知模板库，所有模板须提前审核备案，使用时仅替换变量字段，减少逐条审核负担。

标准化模板须包含：
- 模板编号（如：NTF-LOGIN-001）
- 适用场景描述
- 固定文案内容
- 变量字段说明
- 审核状态与有效期
